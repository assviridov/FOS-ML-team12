# Методические указания преподавателям по использованию ресурсов

Преподаватель заранее предоставляет notebook, данные, индивидуальные варианты и критерии оценивания. Проверка строится по принципу постепенного усложнения: текущая обратная связь на учебных практикумах, рубрикаторная проверка самостоятельных работ и отдельное оценивание итогового проекта и его защиты.
1. До учебного практикума проверить работоспособность notebook и сформулировать ровно два мини-задания.
2. На занятии оценить выполнение мини-заданий по шкале 0–2 и зафиксировать балл в журнале активности.
3. При проверке СР убедиться в соответствии индивидуальному варианту и выполнить Restart Kernel and Run All.
4. Оценивать СР по соответствующей 100-балльной рубрике; отдельно проверить отсутствие data leakage и корректность работы с test.
5. Не требовать презентацию или формальную защиту СР. При сомнении в понимании провести краткую выборочную проверку по notebook.
