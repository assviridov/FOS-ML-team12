# Методические указания обучающимся

До начала работы студент знакомится с форматом конкретного оценочного средства, индивидуальным вариантом, сроком сдачи и рубрикой. Основной принцип: учебные практикумы выполняются на занятии, самостоятельные работы отправляются преподавателю без презентации, а итоговый мини-проект представляется и защищается индивидуально.

1. На учебном практикуме открыть выданный notebook, последовательно изучить обучающий пример и выполнить два мини-задания.
2. Показать результаты мини-заданий преподавателю на занятии и кратко объяснить внесённые изменения.
3. Для СР получить индивидуальный вариант, выполнить notebook самостоятельно и сохранить все результаты выполнения.
4. Перед отправкой СР выполнить Restart Kernel and Run All, проверить относительные пути, графики, таблицы и выводы.
5. Отправить СР № 1–3 в установленный срок. Презентация для этих работ не готовится.
6. Для итогового мини-проекта подготовить notebook или репозиторий и презентацию из 6–8 слайдов.
7. На защите мини-проекта объяснить постановку, данные, модели, метрики, ошибки, ограничения и ключевые фрагменты кода.

| Форма работы | Действия студента | Форма передачи результата |
|--------------|-------------------|---------------------------|
| Учебный практикум | Изучить обучающий пример, выполнить два мини-задания и объяснить результат. | Показать notebook на занятии; отдельная отправка и презентация не требуются. |
| СР № 1–3 | Индивидуально выполнить выданный вариант по шаблону и рубрике. | Отправить выполненный notebook и необходимые CSV; без презентации и формальной защиты. |
| Итоговый мини-проект | Выполнить полный ML-проект, подготовить model card и презентацию. | Отправить проектные материалы и пройти обязательную индивидуальную защиту. |
