# КИМ-2 Самостоятельная работа 2. Сравнительное исследование классических моделей машинного обучения на задачах регрессии и классификации
 

# Самостоятельная работа № 2

## Сравнительное исследование классических моделей машинного обучения на задачах регрессии и классификации

Работа с двумя наборами данных — один для регрессии и один для классификации. Предварительная обработка данных, построение и сравнение линейных моделей, k‑NN, SVM, Decision Tree, Random Forest, Bagging и Gradient Boosting для каждого набора данных, регуляризация, cross‑validation, GridSearchCV / RandomizedSearchCV, подбор гиперпараметров, выбор и оценка лучшей модели.

---

### Основной принцип СР № 2

> Студент обязан применить каждое установленное семейство моделей к обоим датасетам в соответствующей постановке: регрессионную реализацию — к регрессии, классификационную реализацию — к классификации. Нельзя ограничиться разными наборами семейств для двух задач.

---

## Модельный набор для двух датасетов

| Семейство | Регрессионный датасет | Классификационный датасет | Что сравнивается |
|-----------|------------------------|---------------------------|-------------------|
| **Линейные модели** | `LinearRegression`; `Ridge`; `Lasso`; `ElasticNet` | `LogisticRegression` с базовым и при необходимости `balanced class_weight` | Коэффициенты, регуляризация, простота и интерпретируемость |
| **kNN** | `KNeighborsRegressor` | `KNeighborsClassifier` | Scaling, число соседей, метрика расстояния и вычислительная стоимость |
| **SVM** | `SVR`: linear и/или RBF | `SVC`: linear и RBF; `probability`/`decision score` для ROC/PR | Параметры C, gamma, kernel и чувствительность к scaling |
| **Решающие деревья** | `DecisionTreeRegressor` | `DecisionTreeClassifier` | `max_depth`, `min_samples_leaf`, переобучение и интерпретация правил |
| **Ансамбли** | `BaggingRegressor`; `RandomForestRegressor`; `GradientBoostingRegressor` | `BaggingClassifier`; `RandomForestClassifier`; `GradientBoostingClassifier` | Снижение variance, устойчивость, `n_estimators`, `learning_rate` и `feature importance` |

---

## Обязательная последовательность выполнения СР № 2

1. Сформулировать две прикладные задачи, определить target, единицы, положительный класс и пользователей результатов.
2. Выполнить аудит качества каждого датасета и проверить доступность признаков в момент прогноза.
3. Выделить независимый test для каждого датасета.
4. Создать отдельные preprocessing pipeline для моделей, чувствительных и нечувствительных к масштабу.
5. Построить baseline и все обязательные семейства моделей на каждом датасете.
6. Провести единое cross‑validation сравнение внутри каждого датасета по основной и дополнительным метрикам.
7. Выбрать минимум два перспективных семейства на каждом датасете и выполнить `GridSearchCV` или `RandomizedSearchCV`.
8. Зафиксировать финальную модель для каждой задачи до просмотра test.
9. Один раз оценить финальную регрессионную и классификационную модели на test.
10. Выполнить residual analysis для регрессии и FP/FN, threshold и class‑wise analysis для классификации.
11. Сформулировать сравнительный вывод: какие семейства оказались эффективными на каждой структуре данных и почему.

## Обязательные метрики СР № 2

| Задача | Основные метрики | Диагностические материалы |
|--------|------------------|----------------------------|
| **Регрессия** | Основная метрика варианта; MAE; RMSE; R²; median absolute error | Actual vs predicted; распределение остатков; крупнейшие ошибки |
| **Классификация** | Основная метрика варианта; accuracy; precision; recall; F1; balanced accuracy; ROC‑AUC; PR‑AUC | Confusion matrix; ROC/PR; threshold analysis; FP/FN; метрики по классам |
